# PostPNG Maker

PostPNG Maker は、PDFをスマホで扱いやすい PNG/JPEG 画像へ変換するブラウザ内完結型PWAです。X投稿向けの4枚セット作成と、汎用的なPDF画像化の通常モードを備えています。

## プライバシー

選択したPDFと生成画像は外部サーバーへ送信されません。PDF.js、PDF worker、JSZipは `libs/` にローカル同梱する設計で、変換処理はブラウザ内で実行します。現在のリポジトリには取得失敗時のプレースホルダーを置いているため、配布前に下記の公式配布ファイルへ差し替えてください。

> 注意: この環境では外部パッケージ取得がプロキシの 403 Forbidden で失敗したため、`libs/pdf.min.js`、`libs/pdf.worker.min.js`、`libs/jszip.min.js` は手動で公式配布ファイルへ差し替えてください。取得元は以下です。
>
> - `libs/pdf.min.js`: `pdfjs-dist@4.10.38/build/pdf.min.mjs`
> - `libs/pdf.worker.min.js`: `pdfjs-dist@4.10.38/build/pdf.worker.min.mjs`
> - `libs/jszip.min.js`: `jszip@3.10.1/dist/jszip.min.js`

## X投稿モードの使い方

1. 「PDFを選択」からPDFを読み込みます。
2. 横幅と余白トリミングを選びます。
3. 必要に応じて投稿本文・ALT下書きを入力します。
4. 「変換する」を押すと、1〜4ページが「X投稿セット1」、5〜8ページが「X投稿セット2」のように4ページごとに分かれます。
5. 各セットで「セットをZIP保存」「本文をコピー」「ALTをまとめてコピー」が使えます。各ページの個別保存もできます。

## 通常モードの使い方

通常モードでは、PNG/JPEG、JPEG品質、解像度倍率、ページ範囲、背景色、余白トリミングを選択できます。ページ範囲は `1-4`、`1,3,5`、`2-4,8` のように指定します。不正な範囲やPDFページ数を超える指定はエラー表示します。

JPEGでは透明背景を使用できません。JPEG選択時に透明背景が選ばれていた場合は自動で白背景へ変更されます。

## PWA / オフライン利用

初回ロード後、アプリシェルは Service Worker によりキャッシュされ、ホーム画面追加後はPWAとして起動できます。キャッシュ対象はHTML、CSS、JS、ローカルライブラリ、manifest、実在するアイコンのみです。PDFファイルや生成画像はキャッシュしません。

オフライン時はUIの起動と、既にキャッシュ済みのアプリ本体の表示が可能です。ただし、初回ロード前やライブラリ未取得の状態では変換できません。

更新が反映されない場合は、ブラウザのサイトデータ/キャッシュ削除、PWAの再追加、または再読み込みを行ってください。

## Android Chromeでの使い方

1. Android Chromeで `index.html` を開きます。
2. メニューから「ホーム画面に追加」または「アプリをインストール」を選択します。
3. ホーム画面のアイコンから起動します。
4. 画面幅360〜430pxの縦画面を想定し、主要ボタンは44px以上、「変換する」は大きめのボタンとして表示されます。

## 大きいPDFの注意

ページ数が多いPDFや高倍率変換はメモリを多く使います。スマホでは20ページ超の一括変換時に確認を表示します。重い場合は倍率を下げる、ページ範囲を分ける、またはPCで実行してください。

## 制限

- パスワード付きPDFは非対応または制限があります。
- PDFによってはフォント、注釈、透明効果の表示が元PDFと完全一致しない場合があります。
- 透明背景はPNGのみ対応です。JPEGでは白背景に変換します。
- 余白トリミングは白背景・黒背景を検出します。透明やカスタム背景では誤検出防止のため制限があります。
- Clipboard APIが使えない環境では、手動コピー用テキストエリアを表示します。

## ファイル構成

- `index.html`: アプリの画面構造
- `css/style.css`: モバイル優先のUIスタイル
- `js/app.js`: 画面制御、モード制御、変換フロー
- `js/pdf-service.js`: PDF.js読み込みとPDFレンダリング
- `js/image-service.js`: リサイズ、Blob生成、余白トリミング
- `js/export-service.js`: ダウンロード、ZIP、コピー処理
- `js/settings-service.js`: 設定保存、ディープマージ、ページ範囲解析
- `js/pwa-service.js`: Service Worker登録
- `service-worker.js`: PWAアプリシェルキャッシュ
- `manifest.json`: PWAメタデータとアイコン参照
- `icons/`: `icon-192.png`、`icon-512.png`、`maskable-icon-512.png`
- `libs/`: PDF.js、PDF worker、JSZipのローカル配布ファイル

## 使用ライブラリ

- PDF.js / pdfjs-dist: PDFのブラウザ内レンダリング
- JSZip: ZIPファイル生成

## ライセンス

アプリ本体のライセンスはプロジェクトのライセンスに従います。PDF.js と JSZip は各ライブラリのライセンスに従います。配布時は同梱ライブラリのライセンス表記を確認してください。
